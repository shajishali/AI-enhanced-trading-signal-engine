📋 HOW TO START THE APPLICATION
═══════════════════════════════════════════════════════════════

👉 SIMPLE METHOD:
   Just double-click: start_trading_engine.bat
    
   That's it! It will start everything automatically.

───────────────────────────────────────────────────────────────

📝 WHAT GETS STARTED:

   ✅ Celery Worker
      └─ Processes background tasks
      └─ Updates database every 30 minutes
      └─ Generates trading signals automatically

   ✅ Django Web Server
      └─ Runs on http://localhost:8000
      └─ Shows your trading dashboard
      └─ Displays trading signals

───────────────────────────────────────────────────────────────

🛑 TO STOP THE APPLICATION:

   Double-click: stop_trading_engine.bat

   OR manually close both windows:
   - "AI Trading Engine - Celery Worker" window
   - "AI Trading Engine - Django Server" window

───────────────────────────────────────────────────────────────

⚙️ MANUAL START (If needed):

   Open 2 separate command prompts:

   Terminal 1 (Celery):
   └─ python -m celery -A ai_trading_engine worker -B --loglevel=info --pool=solo

   Terminal 2 (Django):
   └─ python manage.py runserver 0.0.0.0:8000

───────────────────────────────────────────────────────────────

📊 WEB INTERFACE:

   Once started, open your browser to:
   http://localhost:8000

───────────────────────────────────────────────────────────────

⏰ AUTOMATIC UPDATES:

   The system will automatically:
   - Update market data every 30 minutes
   - Generate new trading signals every 30 minutes
   - Keep your database fresh
   - Display latest signals on dashboard

───────────────────────────────────────────────────────────────

⚠️  IMPORTANT NOTES:

   1. Keep both service windows open at all times
   2. Don't close the Celery Worker window
   3. Don't close the Django Server window
   4. If you close them, just run start_trading_engine.bat again

───────────────────────────────────────────────────────────────

🐛 TROUBLESHOOTING:

   Problem: "Port 8000 already in use"
   Solution: Close any other Django instances or change the port

   Problem: "Celery worker not running"
   Solution: Check if Celery is installed: pip list | findstr celery

   Problem: "Can't connect to database"
   Solution: Make sure PostgreSQL or your database is running

───────────────────────────────────────────────────────────────

✅ READY TO START?

   Just double-click: start_trading_engine.bat
